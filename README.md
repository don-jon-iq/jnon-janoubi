# جنون جنوبي — Southern Obsession

صفحة بيع تفاعلية باللغة العربية لمنتجات تراث البصرة (دبوس، ميدالية مينا، مغناطيس).

موقع ثابت بالكامل — لا يحتاج أي تنصيب أو خادم. افتح `index.html` مباشرة، أو انشره عبر Docker / Dokploy (انظر قسم النشر أدناه)، أو ارفع المجلد على أي استضافة ثابتة (Netlify / Vercel / GitHub Pages).

## قبل النشر — خطوة واحدة إلزامية

افتح [js/config.js](js/config.js) وعدّل:

1. **رقم الواتساب** — استبدل `PUT_YOUR_NUMBER_HERE` برقمك بالصيغة الدولية بدون `+`:
   ```js
   WHATSAPP_NUMBER: '9647701234567',
   ```
2. **الأسعار** — كل منتج له سطر `price` (بالدينار العراقي).
3. **الأوصاف والأسماء** — عدّلها بحرية من نفس الملف.

## كيف يعمل الطلب

1. الزبون يختار المجموعة (تراث البصرة).
2. يختار القطع والكميات.
3. يكتب اسمه ورقمه وعنوانه — ثم يضغط «إرسال الطلب عبر واتساب».
4. تنفتح محادثة واتساب معك وفيها رسالة جاهزة بكل تفاصيل الطلب والمجموع.

## النشر على Dokploy (السيرفر الخاص)

المشروع جاهز كحاوية Docker تعتمد على nginx. عندك طريقتان في Dokploy:

### الطريقة الأولى — Application + Dockerfile (الأبسط، موصى بها)

1. في لوحة Dokploy اضغط **Create → Application**.
2. **Provider:** GitHub، واختر مستودع هذا المشروع والفرع `main`.
3. **Build Type:** اختر **Dockerfile** (المسار: `Dockerfile`).
4. في تبويب **Domains** أضف نطاقك، ووجّهه إلى **المنفذ `80`**، وفعّل **HTTPS / Let's Encrypt**.
5. اضغط **Deploy**. كل `git push` لاحق يعيد النشر تلقائياً إذا فعّلت **Auto Deploy / Webhook**.

### الطريقة الثانية — Docker Compose

1. **Create → Compose**، واربطه بمستودع GitHub (يستخدم `docker-compose.yml` الموجود).
2. في **Domains** اربط النطاق بالخدمة `web` على المنفذ `80`.
3. **Deploy**.

### تشغيل محلي بدون Dokploy (للتجربة)

```bash
docker compose up -d --build
# ثم افتح: http://localhost:8123
```

> داخل الحاوية nginx يعمل على المنفذ **80** (هذا ما يوجّه إليه Dokploy النطاق).
> المنفذ **8123** هو منفذ المضيف للتشغيل المباشر خارج Dokploy فقط.

> ملاحظة: عدّل `js/config.js` (رقم الواتساب والأسعار) **قبل** النشر — الملف `index.html` يُقدَّم بدون تخزين مؤقت، فأي تعديل ينعكس فوراً بعد إعادة النشر.

## ملفات النشر

| الملف | الوظيفة |
|---|---|
| `Dockerfile` | بناء صورة nginx وتقديم الموقع |
| `nginx.conf` | إعدادات الخادم (ضغط، تخزين مؤقت، دعم الفيديو، أمان) |
| `docker-compose.yml` | تعريف الخدمة لـ Dokploy / docker compose |
| `.dockerignore` | استبعاد ملفات التطوير من الصورة |
