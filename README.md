# جنون جنوبي — Southern Obsession

متجر تفاعلي باللغة العربية لمنتجات تراث البصرة (دبوس، ميدالية مينا، مغناطيس)، مع **لوحة تحكم آمنة** لإدارة المجموعات والقطع والإعدادات.

التطبيق الآن **خادم Node + Express + SQLite** يقدّم الموقع العام، لوحة التحكم `/admin`، الصور المرفوعة، وواجهة API. البيانات تُحفظ في قاعدة SQLite على قرص دائم، فأي تعديل من لوحة التحكم يظهر فوراً على المتجر بدون إعادة نشر.

## لوحة التحكم `/admin`

افتح `https://<نطاقك>/admin` وسجّل الدخول بكلمة المرور (المتغير `ADMIN_PASSWORD`). من اللوحة تقدر:

- إضافة / تعديل / حذف **المجموعات** و**القطع**، وإظهارها أو إخفاؤها، وإعادة ترتيبها.
- تعديل **إعدادات المتجر**: رقم الواتساب ورمز العملة.
- **رفع صور** المنتجات (JPG · PNG · WEBP · GIF حتى ٥ ميغابايت) أو إعادة استخدام صور سابقة.

> لم تَعُد تُعدّل `js/config.js` يدوياً — كل شيء صار من لوحة التحكم.

## المتغيرات المطلوبة (أسرار)

| المتغير | الوصف |
|---|---|
| `ADMIN_PASSWORD` | كلمة مرور الدخول للوحة التحكم. تُجزّأ (hash) في الذاكرة عند الإقلاع. |
| `SESSION_SECRET` | نص عشوائي طويل لتوقيع جلسة الدخول. |
| `NODE_ENV` | `production` على السيرفر. |

> توليد سرّ عشوائي: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`.
> اضبط هذه القيم في Dokploy ضمن **Environment / Secrets** — لا تضعها في الكود ولا في git.

## كيف يعمل الطلب

1. الزبون يختار المجموعة، ثم القطع والكميات، ثم يكتب بياناته.
2. يضغط «إرسال الطلب عبر واتساب» → تنفتح محادثة واتساب معك برسالة جاهزة فيها كل تفاصيل الطلب والمجموع. الدفع عند الاستلام (لا توجد بوابة دفع).

## النشر على Dokploy (السيرفر الخاص)

### الطريقة الأولى — Application + Dockerfile (موصى بها)

1. **Create → Application**، واختر مستودع GitHub والفرع `main`.
2. **Build Type:** `Dockerfile`.
3. **Environment:** أضف `ADMIN_PASSWORD` و`SESSION_SECRET` و`NODE_ENV=production`.
4. **Volumes / Mounts:** أضف حجماً دائماً (persistent volume) على المسار **`/app/data`** — هنا تُحفظ قاعدة البيانات والصور المرفوعة، وتبقى محفوظة عبر كل إعادة نشر.
5. **Domains:** أضف نطاقك ووجّهه إلى **المنفذ `3000`**، وفعّل **HTTPS / Let's Encrypt**.
6. **Deploy**. فعّل **Auto Deploy** ليُعاد النشر مع كل `git push`.

### الطريقة الثانية — Docker Compose

1. **Create → Compose** من مستودع GitHub (يستخدم `docker-compose.yml`).
2. اضبط الأسرار، واربط الحجم الدائم بـ `/app/data`، واربط النطاق بالخدمة `app` على المنفذ `3000`.
3. **Deploy**.

### تشغيل محلي للتجربة

```bash
# عبر Docker (موصى به — يبني better-sqlite3 داخل الحاوية)
ADMIN_PASSWORD=secret SESSION_SECRET=dev-secret docker compose up -d --build
# ثم افتح: http://localhost:8123  (واللوحة على /admin)

# أو مباشرة بـ Node (يتطلب أدوات بناء لـ better-sqlite3)
npm install
ADMIN_PASSWORD=secret SESSION_SECRET=dev-secret COOKIE_SECURE=false npm start
# ثم افتح: http://localhost:3000
```

> عند أول إقلاع تُنشأ قاعدة البيانات وتُعبّأ تلقائياً بالكتالوج الحالي (لا تفقد أي شيء).
> داخل الحاوية يستمع Node على المنفذ **3000** (هذا ما يوجّه إليه Dokploy). المنفذ **8123** للتشغيل المحلي فقط.

## نسخ احتياطي

كل بيانات المتجر (المنتجات + الصور) في الحجم الدائم `/app/data`. انسخ احتياطياً بشكل دوري:
الملفات `store.db*` ومجلد `uploads/` (أو خذ snapshot للحجم من Dokploy).

## الأمان

- الدخول بكلمة مرور واحدة مجزّأة (bcrypt)، وجلسة موقّعة (JWT) في كوكي `httpOnly + Secure + SameSite=strict` صالحة ساعتين.
- تحديد معدّل محاولات الدخول، حماية CSRF (double-submit)، ترويسات helmet/CSP.
- رفع الصور يُتحقق من نوعها الحقيقي (magic bytes) وحجمها، وتُخزّن بأسماء عشوائية آمنة.
- كل ذلك خلف HTTPS من Dokploy/Traefik.

## بنية المشروع

| المسار | الوظيفة |
|---|---|
| `index.html`, `css/`, `js/` | المتجر العام (الواجهة) |
| `js/config.js` | يجلب الكتالوج من `/api/store` |
| `admin/` | واجهة لوحة التحكم (HTML/CSS/JS) |
| `server/` | خادم Express: API، قاعدة البيانات، المصادقة، الرفع |
| `server/db/seed.js` | تعبئة الكتالوج الأولي عند أول إقلاع |
| `assets/` | الشعار والفيديو والصور المبدئية |
| `Dockerfile`, `docker-compose.yml` | النشر (Node + حجم دائم) |
